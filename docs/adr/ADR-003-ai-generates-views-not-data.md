# ADR-003: AI generates Views, not Data

- Status: Accepted
- Date: 2026-07-02
- Related: PRD §4.5 AI as Reader, §9 Long-term Design Goals, [DATA_MODEL.md §8 View](../DATA_MODEL.md#8-view)

## Decision

AI는 원본 데이터(Log, 사진, 메시지, 메타데이터)를 생성하거나 수정하지 않는다.

AI는 축적된 데이터로부터 **View**(회상, 추천, 연결, 요약)만 생성하며, 모든 View는 원본에서 언제든 재생성 가능한 파생물로 취급한다.

## Context

Duri는 AI 모델이 여러 번 교체될 만큼 긴 수명을 목표로 한다. AI의 출력이 원본 데이터와 섞이면, 특정 모델의 품질·편향이 데이터 자산에 영구적으로 새겨지고 Human Readable First(ADR-001) 원칙이 훼손된다.

## Alternatives Considered

1. **AI가 데이터를 직접 구조화·수정** — 즉각적인 정리 품질은 좋아 보이지만, 원본 훼손 위험과 모델 종속이 발생한다.
2. **AI 출력물을 원본과 동급으로 저장** — 회상·요약을 영구 보존할 수 있지만 원본과 파생물의 경계가 무너져 장기적으로 무엇이 사실인지 알 수 없게 된다.
3. **AI 미사용(순수 검색만)** — 안전하지만 Vault 기반 회상·추천이라는 핵심 가치를 포기하게 된다.

## Reason

- 원본 불변성이 보장되어야 10년 뒤에도 데이터를 신뢰할 수 있다.
- AI 모델이 더 좋아지면 같은 원본에서 더 좋은 View를 다시 만들 수 있다 — AI 품질이 데이터 가치에 복리로 작용한다.
- View를 파생물로 취급하면 캐시처럼 자유롭게 버리고 재생성할 수 있어 시스템이 단순해진다.

## Consequences

- (+) AI 교체·업그레이드가 데이터에 아무 영향을 주지 않는다.
- (+) 저장 계층이 원본 스토어와 파생 스토어로 명확히 분리된다.
- (−) 사용자가 아낀 특정 요약·회상을 보존하고 싶다면, 그것을 명시적으로 새로운 Log(원본)로 승격하는 별도 흐름이 필요하다.
- View 재생성 비용(AI 호출)이 반복 발생할 수 있으므로 캐싱 전략이 필요하다.
